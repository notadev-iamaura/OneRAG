# Support

Use the right channel so the issue can be handled quickly.

## Questions and Usage Help

Open a GitHub issue with:

- What you are trying to build.
- Your deployment mode, such as local CLI, Docker quickstart, API server, or
  frontend.
- Relevant environment details, package manager versions, and logs.
- A minimal reproduction using sample or dummy data.

## Bugs

Use the bug report template and include:

- Expected behavior.
- Actual behavior.
- Steps to reproduce.
- Commands already run.
- Whether the issue affects backend, frontend, Docker, docs, CI, or quickstart.

## Feature Requests

Use the feature request template. Explain the RAG workflow, provider,
integration, or user problem the feature should solve.

## Security Issues

Do not open public issues containing vulnerability details, secrets, private
documents, exploit payloads, or production logs. Follow `SECURITY.md` instead.

## Maintainer Expectations

Response time depends on project load and severity. Security reports and
release-blocking regressions are prioritized ahead of routine support questions.
